# Codebook — the `#inpeer` coding tree (all levels)

This codebook documents **every code** in the vault's hierarchical coding tree: 455 codes (330 directly applied; 125 parent nodes implied by deeper applications), 1,136 total applications, organized as one numbered table per hierarchy level (Tables 1–11). It follows qualitative-data-analysis codebook conventions: each entry gives the **code name**, a **definition**, the **research question** it serves, its **instance count**, and a **specific grounded example** linked to the vault note where the code was applied.

> **AI-use note:** compiled by Claude Fable 5 (2026-07-05) exclusively from this vault's files, following the disclosure practice in [README.md](README.md); checked by Jay Patel. Wherever the researcher wrote an inline gloss beside an applied code (the "*This means that…*" annotations in memos and captures), that gloss is quoted verbatim as the definition, with the source note cited. Definitions without a gloss citation were drafted from the code path and its coded excerpts. Counts are a snapshot of the live vault (171 notes) and drift as the vault is edited.

**How to read the tables.**

- **Code** — the full nested tag path. Deeper levels inherit the meaning of their ancestors.
- **Definition** — verbatim researcher gloss (cited) where one exists; otherwise drafted from usage.
- **RQ** — the research question the branch serves (Patel & Chan 2026): **RQ1** *Who engages in informal peer reviewing?* (`who`, `why`); **RQ2** *Where do informal reviewers work?* (`evdInfra`, except `impacts`); **RQ3** *How do reviewers evaluate research?* (`how`); **RQ4** *What are the impacts?* (`evdInfra/impacts`); **Method** = the researcher's participant observation (`digitalEthnography`).
- **Inst.** — applications of the code *or any of its descendants* across the vault.
- **Example** — a verbatim coded excerpt with a `[[wikilink]]` to the note where the code was applied. Parent nodes never applied directly point to a descendant's note.
